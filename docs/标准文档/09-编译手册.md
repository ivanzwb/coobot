# BiosBot 编译手册

## 1. 文档目的

本文档用于说明 BiosBot 当前仓库的安装、编译、运行和基本排查方式。
本文档主要支撑开发环境搭建、本地调试、联调验证和基础问题排查，内容基于当前仓库中的 `backend/package.json`、`web/package.json` 和现有目录结构整理。
本文档重点回答“项目如何启动、前后端如何编译运行、常见问题如何快速排查”。

## 2. 目录结构

```text
biosBot/
  backend/
  web/
  app/
  database/
  产品描述.md
```

当前编译手册只覆盖：

- `backend/`
- `web/`

## 3. 环境要求

- Node.js 18 或更高版本。
- npm。
- 本地可用的 TypeScript 运行环境由项目依赖提供。

如需运行 Skills 中的脚本工具，建议同时具备：

- Python 3。
- Bash 或兼容 shell 环境。

## 4. 后端编译与运行

后端目录：`backend/`

### 4.1 安装依赖

```bash
cd backend
npm install
```

### 4.2 开发运行

```bash
cd backend
npm run dev
```

对应脚本：

- `dev`: `npx ts-node src/index.ts`

### 4.3 编译

```bash
cd backend
npm run build
```

对应脚本：

- `build`: `tsc`

### 4.4 生产启动

```bash
cd backend
npm run start
```

对应脚本：

- `start`: `node dist/index.js`

### 4.5 数据库初始化

```bash
cd backend
npm run migrate
npm run seed
```

对应脚本：

- `migrate`: `npx ts-node src/infra/db/migrate.ts`
- `seed`: `npx ts-node src/infra/db/seed.ts`

## 5. 前端编译与运行

前端目录：`web/`

### 5.1 安装依赖

```bash
cd web
npm install
```

### 5.2 开发运行

```bash
cd web
npm run dev
```

对应脚本：

- `dev`: `vite`

### 5.3 构建

```bash
cd web
npm run build
```

对应脚本：

- `build`: `tsc -b && vite build`

### 5.4 预览

```bash
cd web
npm run preview
```

对应脚本：

- `preview`: `vite preview`

## 6. 推荐启动顺序

### 6.1 本地开发

终端一：

```bash

```


## 7. 运行依赖说明

- 后端依赖本地数据库和文件存储。
- 后端通过外部模型 API 提供推理能力，实际运行前需要准备模型配置和密钥。
- 如需启用 Skill 脚本或 MCP / 脚本工具，需要保证相关运行环境可用。

## 8. 常见问题

### 8.1 后端启动失败

检查顺序：

- Node.js 版本是否符合要求。
- 依赖是否完整安装。
- 模型配置和本地数据库初始化是否完成。
- TypeScript 编译错误是否存在。

### 8.2 前端构建失败

检查顺序：

- `npm install` 是否成功。
- TypeScript 类型是否一致。
- Vite 配置和依赖版本是否匹配。

### 8.3 Skills 或工具不可用

检查顺序：

- Skill 是否存在于 `backend/skills/`。
- Agent 是否引用了对应 Skill。
- Skill 是否经过 `use_skill` 激活。
- 脚本运行环境是否可用。

## 9. 注意事项

- 当前仓库未提供统一根级一键启动脚本，前后端需分别启动。
- 当前手册依据仓库脚本整理，未在本次文档拆分过程中重新执行完整构建验证。

## 10. 编译手册结论

本手册的目标是让开发者能够按当前仓库结构完成安装、启动、联调和基础排查，后续如构建脚本变化应同步更新本手册。
